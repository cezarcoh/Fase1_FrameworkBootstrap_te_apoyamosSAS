Términos Web

1. Hosting


Cuando se contrata un servicio de hosting, se alquila acceso a un servidor web que sirve para publicar contenido en internet, allí se tiene acceso espacio y herramientas que integran y despliegan nuestro contenido, paneles de configuración de éste, de dominios, bases de datos y certificados SSL.

Tipos de distribución o servicio a escoger:

  • Hosting compartido: Varios sitios en el mismo servidor. Limitado pero económico.
  • VPS: Es un servidor físico dividido en entornos virtuales independientes. Tienes tu propio entorno virtual. Más control.

    Uso. Aplicaciones medianas, sitios con tráfico moderado, entornos de staging.

  • Cloud Hosting: Infraestructura distribuida en múltiples servidores. Escalable y moderna.

    Uso. Aplicaciones modernas, microservicios, APIs RESTful.

  • Hosting Dedicado: Un servidor físico completo un solo cliente.

    Uso. Aplicaciones críticas, grandes bases de datos, procesamiento intensivo.

    Para tener en cuenta:

  • Escalabilidad
  • Seguridad
  • Monitoreo

2. Dominio


Un dominio es una entrada en el sistema DNS que traduce nombres legibles (misitio.com) a direcciones IP. Es parte esencial del enrutamiento web. sirve para identificar un sitio web o un servicio en internet. Son el componente legible que reemplaza a las direcciones IP numéricas y complejas, permitiendo a los usuarios acceder a páginas web y servicios de forma sencilla

Funcionamiento

Cuando el usuario teclea un dominio en el navegador web, el navegador consulta el DNS o sistema de nombres de dominio y lo traduce a una IP numérica que los servidores utilizan para conectar al servidor donde se encuentra alojado y así mostrar el sitio web.

Registros DNS principales:

  • A/AAAA: Mapeo a IP IPv4/IPv6
  • CNAME: Alias canónico para subdominios.
  • MX: Servidores de correo.
  • TXT: Metadatos (SPF, DKIM, DMARC).
  • NS: Servidores de nombres autoritativos.

3. Diseño Responsive


El diseño responsive es una metodología de desarrollo que utiliza grids fluidos, media queries CSS y contenido flexible para crear interfaces que se adaptan dinámicamente a diferentes viewports o tamaños de pantalla de cualquier dispositivo. Implementa el principio "mobile-first" con progressive enhancement que es diseñar para dispositivos móviles y luego ir implementando mejoras en diseño o funcionalidades para pantallas de mayor tamaño.

En esencia se diseña basado en puntos específicos de resolución para que el contenido se adapte adecuadamente al espacio disponible priorizando en algunos elementos como el menú, el contenido y los elementos gráficos, así mismo escalando las imágenes y fuentes.

El diseño responsive permite la usabilidad ya que las personas pueden navegar en la información de manera intuitiva ofreciendo una experiencia uniforme en cualquier dispositivo favoreciendo el SEO y los tiempos de programación.
